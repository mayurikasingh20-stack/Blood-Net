FRONTEND API GUIDE - Blood Net / NOVARED
=========================================

Base URL: http://127.0.0.1:5000/api

All JWT-protected endpoints require header: Authorization: Bearer <token>

---

AUTH (/api/auth)
================

POST /api/auth/register  (No Auth)
  Registers a new user. If role=donor, also creates donor profile.

  Body (JSON):
    first_name       string  required
    last_name        string  required
    email            string  required  (unique)
    phone            string  required  (unique)
    password         string  required  (min 6 chars)
    role             string  required  donor | patient | blood_bank | admin
    gender           string  required  Male | Female | Other
    dob              string  required  YYYY-MM-DD
    city             string  required
    address          string  optional

  Donor-only fields:
    blood_group              string  required if role=donor  A+|A-|B+|B-|AB+|AB-|O+|O-
    weight                   number  required if role=donor
    last_donation_date       string  optional  YYYY-MM-DD
    has_chronic_condition    bool    optional
    on_medication            bool    optional
    available                bool    optional  default true

  Response 201: { "message": "registered successfully" }
  Response 400: { "message": "...", "missing_fields": [...] }

---

POST /api/auth/login  (No Auth)
  Returns JWT token. Donor logs in with phone; others with email.

  Blood bank login is BLOCKED (401) if profile status is pending or rejected.

  Body for donor:
    { "phone": "+911234567890", "password": "...", "role": "donor" }

  Body for others:
    { "email": "user@example.com", "password": "...", "role": "blood_bank" }

  Response 200:
    {
      "access_token": "jwt...",
      "name": "First Last",
      "user": { "id", "first_name", "last_name", "email", "phone", "role" }
    }
  Response 401 (blood bank pending):
    { "message": "Your blood bank registration is pending admin approval. You will be able to log in once approved." }
  Response 401 (blood bank rejected):
    { "message": "Your blood bank registration has been rejected. Reason: ..." }

---

GET /api/auth/profile  (JWT)
  Returns current user info.

  Response: { "id", "first_name", "last_name", "email", "phone", "role", "gender", "city" }

---

PATCH /api/auth/profile  (JWT)
  Update profile fields. All fields optional.

  Body (JSON): first_name, last_name, email, phone, city

  Response: { "message": "Profile updated successfully" }

---

POST /api/auth/change-password  (JWT)
  Body: { "current_password": "...", "new_password": "..." }  (new min 6 chars)

  Response: { "message": "Password changed successfully." }

BLOOD BANK (/api/blood-bank)
=============================

POST /api/blood-bank/register  (JWT + blood_bank role)
  Register blood bank facility details. Status set to "pending".

  Body (JSON):
    facility_name     string  required
    contact_person    string  required
    address           string  required
    operating_hours   string  optional  e.g. "9 AM - 6 PM"
    website           string  optional
    available_24x7    bool    optional  default false

  Response 201: { "message": "Blood bank registered successfully. Waiting for admin verification." }

---

GET /api/blood-bank/profile  (JWT + blood_bank role)
  Response:
    { "id", "user_id", "facility_name", "verification_status",
      "verified_at", "rejection_reason", "contact_person",
      "address", "operating_hours", "available_24x7",
      "website", "created_at", "updated_at" }

---

PUT /api/blood-bank/profile  (JWT + blood_bank role)
  Update profile. Same fields as register (all optional).

  Response: { "message": "Blood bank profile updated successfully." }

---

GET /api/blood-bank/dashboard  (JWT + blood_bank role)
  Response:
    { "verification_status", "total_requests", "active_requests",
      "completed_requests", "total_units_requested",
      "total_units_fulfilled" }

PATIENT (/api/patient)
======================

All endpoints require JWT + patient role.

POST /api/patient/register
  Multipart form:
    blood_group_needed   string  required
    hospital_name        string  required
    condition_description string  required
    relation_to_patient  string  required  Self | Family | Friend
    urgency_level        string  optional  Critical | High | Moderate

    "total_units_requested", "total_units_received" }

ROLE MAPPING UTILITY
====================

Backend uses snake_case roles: blood_bank, donor, patient, admin
Frontend UI uses: bloodbank, donor, patient, admin

export const uiRoleToBackendRole = (role) => role === "bloodbank" ? "blood_bank" : role;
export const backendRoleToUiRole = (role) => role === "blood_bank" ? "bloodbank" : role;
export const dashboardPathForRole = (role) => {
  if (role === "blood_bank" || role === "bloodbank") return "/bloodbank";
  if (role === "admin") return "/admin";
  return `/${role}-dashboard`;
};

BLOOD BANK REGISTRATION FLOW
=============================

Step 1: POST /auth/register  → creates User with role=blood_bank
Step 2: POST /auth/login     → gets JWT (allowed because no profile yet)
Step 3: POST /blood-bank/register → creates BloodBank with status=pending
Step 4: User redirected to login → login BLOCKED with "pending admin approval"
Step 5: Admin approves via PATCH /admin/blood-banks/:id/approve
Step 6: Blood bank can now login and access dashboard
